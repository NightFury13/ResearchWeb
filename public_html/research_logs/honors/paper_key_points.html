Reading & Discussing

Conceptual verification of adversarial pipeline


> Intriguing Properties of Neural Networks - Szegedy et al. :

+ The smoothness assumption that is assumed by many kernel methods doesn't hold true for deep learning tasks!

The deep stack of non-linear layers in between the input and output units of the DNN are a way for the network to encode a non-local generalization prior [ x!~y but phi(x)~phi(y) ].
So, it is possible for the output units to assign non-zero (presumably non-epsilon) probabilities to region of the input space that contain no training examples in their vicinity. These areas can be thought of as representing the same object/view from a different orientation/rotation/view-points etc.


> Explaining and Harnessing Adversarial Examples - Goodfellow et al. :

+ Linear behaviour in the high-dimensional spaces is sufficient to cause adversarial examples!

The reasons for the existence of adversarial examples was unknown and speculations suggested high-nonlinearity of the model along with insufficient regularization as the cause for it. These guys showed how simple linear behaviour in a high-dimensional space of data was enough to create adversarial examples.
Normally we expect the classification tasks to work well in case of perturbations of magnitude lesser than the precision of our features (eg. 0-255 pixel range), so a 120.2 should be considered as a 120 pixel value. However, the linear nature of the model causes a problem here. Consider the dot product of the weight matrix(W) with an adversarial example(x' = x+nu),

W.x' = W.x + W.nu

The adversarial examples hence cause an increase of W.nu rather than a negligible nu. This causes the unexpected increase/decrease in the target class score. Hence, a simple linear model can have adversarial examples if its input has sufficient dimensionality.
