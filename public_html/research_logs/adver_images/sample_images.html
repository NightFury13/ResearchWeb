Adversarial Image Generation

Correcting Adversarial Image Generation Pipeline

The adversarial image generation routines used previously showed no class-specific noise behaviour and this led me to discover that infact there was a bug in the generation of fooling images. The correct process goes about like this -
> Set the "force_backward: true" parameter in the network configuration prototxt file.
 > Compute the forward pass of raw image through the network.
 > Set the "diff" of the last fully-connected layer to 1 at the index i where i is the ground-truth class.
