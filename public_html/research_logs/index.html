OCR works on fixed dimensional Images6-13th Sept'16

Evaluation of the RNN+CTC modules in smaller steps Done

The RNN+CTC model (simple LSTM+CTC, not bi-directional) trains very well using raw image grayscale inputs of fixed dimensions (height X widht) for OCR (digits recognition) task. The training method however doesn't scale well at all for Scene-Text tasks of varying image widths (different sequence lengths).

RNN's fail to learn OCR/Scene-Text9-16th Aug'16

Evaluation of the RNN+CTC modules Halted

The outputs of the models for OCR as well as Scene-Text tasks are coming out to be repetition of a single character. Unable to locate the bug causing these anamolous behaviour but sure that it is a bug as even the END-OF-SEQUENCE character is not being learnt by the model (does not appear in the predictions). 

* Find and fix this bug *

Since I've been unable to rectify this bug for quite some time now. I plan on writing code based on a CPU-compatible widely used CTC implementation. I want to make sure that the way I prepare and feed the data to the RNN is correct.

  >> If the bug is found in data-preparation routines, it would be a small task replicating the fix in our current codes.

  >> If the bug isn't found in the data-prep routines, it would imply a bug in either the CTC-library itself or in the way I am using it. In either case, this would require understanding the CTC-library code in depth and rectifying the issue.
